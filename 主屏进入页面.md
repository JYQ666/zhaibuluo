# 定制衣柜案例展示网页 — 完整需求规格

## 一、项目概述

你是一名前端开发工程师。请根据以下需求，开发一个**定制衣柜案例展示网页**。

**核心场景**：一位定制衣柜商家，希望客户通过手机扫描一个通用二维码，打开一个网页，浏览该商家按小区分类的实际案例效果图。

**技术约束**：
- 纯静态网站，部署在 GitHub Pages 上，完全免费
- 不需要后端服务器
- 配套一个本地管理界面（HTML 页面），用于添加/管理案例，管理界面操作后生成 JSON 配置文件，通过 git push 更新到 GitHub Pages
- 图片存放在项目仓库内（`/images` 目录），通过相对路径引用

---

## 二、二维码策略

- **一个通用二维码**，所有客户扫同一个码，进入同一个网页
- 二维码指向 GitHub Pages 的网址（如 `https://<username>.github.io/<repo>/`）

## 三、设计方向与品牌定位

### 3.0 品牌美学方向

**核心定位**：不是"展示案例"，而是"展示空间改造的艺术"。

**美学参考**：
- **建筑杂志风格**（如 Domus、El Croquis）：大图、留白、排版精致
- **高端地产营销**：沉浸式、情感化、强调生活方式
- **拒绝**：SaaS 灰、卡片网格、渐变文字、小大写标签

**品牌个性三词**：精致 · 克制 · 有温度

**色彩策略**：**Committed（单色主导）**
- 主色：深墨绿 `oklch(25% 0.08 160)` 或深藏蓝 `oklch(25% 0.1 240)` —— 传达精致、专业、有质感
- 背景：纯白 `oklch(98% 0 0)` 或极浅灰 `oklch(96% 0.005 160)` —— 让效果图成为主角
- 强调色：暖铜色 `oklch(65% 0.15 60)` —— 用于按钮、悬停状态、关键元素
- 文字：深灰 `oklch(20% 0.02 160)` —— 避免纯黑，保持温度

**字体选择**（避开 AI slop 列表）：
- 中文标题：思源宋体（Noto Serif SC）或霞鹜文楷（LXGW WenKai）—— 精致、有文化感
- 中文正文：思源黑体（Noto Sans SC）或苹方（PingFang SC）—— 清晰、现代
- 英文装饰字体（仅用于 Logo 或极少量英文点缀）：Playfair Display
- **拒绝**：Inter、Roboto、Arial、system fonts、IBM Plex Sans

## 四、页面结构与内容

网页为**单页应用（SPA）**，采用**编辑式长页面**布局，从上到下分为以下区域：

### 4.1 品牌区（Hero Section）

**目标**：3 秒内传达"精致、专业、有温度"。

**布局**：
- 全屏高度，背景为**单张精选案例大图**（非渐变、非粒子）
- 图片处理：轻微暗角 + 低饱和度，让文字可读
- 内容居中：Logo + 品牌名 + 一句话价值主张（非 slogan，而是"我们做什么"）

**文案示例**：
- 品牌名：「你的品牌名」
- 价值主张：「为每个家庭，定制专属收纳空间」

**动效**（克制、有意图）：
- 页面加载：图片从模糊到清晰（0.8s），文字从下方淡入（0.6s，延迟 0.2s）
- 滚动提示：底部一个细长的向下箭头，缓慢上下浮动（非呼吸动画）
- **无**视差滚动（视差容易让图片失真，且与"精致"冲突）

**移动端**：
- 图片裁切为竖版构图，文字缩小但保持可读

### 4.2 精选案例区（Featured Section）

**目标**：用 3-5 个最出彩的案例，建立"这个商家很专业"的信任。

**布局**：**全屏横向滚动**（非卡片滑动）
- 每个案例占据整个视口（100vw × 100vh）
- 左侧：大图（占 70% 宽度）
- 右侧：案例信息（占 30% 宽度）：小区名 + 户型 + 一句话亮点 + 图片数量指示

**交互**：
- 横向滚动：鼠标滚轮 / 触摸滑动 / 底部进度条拖拽
- 每个案例内多图：点击大图进入灯箱，左右切换
- **无**悬停效果（全屏状态下悬停无意义）

**动效**：
- 案例切换：图片从右侧滑入（0.5s），文字淡入（0.4s，延迟 0.1s）
- 使用 GSAP ScrollTrigger 实现横向滚动

**移动端**：
- 改为纵向全屏滑动（每个案例一屏）
- 图片占满全屏，信息浮在底部（半透明黑底白字）

### 4.3 小区选择区（Community Section）

**目标**：快速找到"我的小区"，建立"原来他们也做过我们小区"的亲切感。

**布局**：**非对称网格**（非均等卡片）
- 大小区（案例数 ≥ 5）：占据 2 列，显示案例数 + 1 张代表图
- 小小区（案例数 < 5）：占据 1 列，只显示小区名 + 案例数
- 使用 CSS Grid：`grid-template-columns: repeat(auto-fit, minmax(200px, 1fr))`

**视觉**：
- 背景：浅灰 `oklch(96% 0.005 160)`，与品牌区区分
- 小区名：深灰文字，无装饰
- 案例数：暖铜色小字，放在小区名下方
- **无**卡片边框、阴影、圆角（保持克制）

**交互**：
- 搜索框：顶部固定，输入时实时过滤
- 点击小区：平滑滚动到该小区的案例区（锚点跳转）

**移动端**：
- 改为单列列表，搜索框置顶

### 4.4 案例详情区（Case Detail）

**目标**：以案例为单位沉浸式浏览，每个案例是一个完整的"家"。

**布局**：**案例卡片纵向排列**（非瀑布流、非轮播）
- 点击小区后，展示该小区的所有案例
- 每个案例占据一个独立区域，之间有充足间距（80-120px）
- 每个案例包含：
  - 案例标题（如"翡翠湖畔 3栋 1201室"）
  - **多张图片横向轮播**（Swiper.js，支持手势滑动）
  - 图片尺寸：宽度占满容器，高度 60vh（桌面端）/ 50vh（移动端）
  - 底部有分页指示点

**交互**：
- 图片轮播：支持触摸手势滑动 + 左右箭头
- 点击任意图片：进入灯箱模式（全屏查看、缩放、左右切换）
- 案例之间有分隔（用间距而非线条分隔）

**动效**：
- 案例进入视口：淡入 + 轻微上浮（0.4s）
- 使用 Intersection Observer 触发

**移动端**：
- 与桌面端保持一致：以案例为单位纵向排列，每个案例一个轮播
- 图片宽度 100%，高度 50vh
- 轮播支持触摸手势滑动

### 4.5 底部联系区（Footer Section）

**目标**：简单直接，让客户能联系到你。

**布局**：
- 背景：深墨绿 `oklch(25% 0.08 160)`
- 左侧：品牌 Logo + 一句话价值主张
- 右侧：微信二维码（白底，尺寸 160×160px）+ "扫码添加微信" 文字

**移动端**：
- 垂直居中，二维码在上，品牌信息在下

## 四、数据结构

案例数据存储在 `/data/cases.json` 文件中，结构如下：

```json

    "slogan": "一句话 slogan"

  "wechat_qrcode": "images/brand/wechat-qrcode.png",
  "communities": [
    {
      "id": "community-001",
      "name": "翡翠湖畔",
      "cases": [

            "images/community-001/case-001/2.jpg",

          ]
        },

          "featured": false,
          "images": [
            "images/community-001/case-002/1.jpg",
            "images/community-001/case-002/2.jpg"
          ]

      ]
    },
    {
      "id": "community-002",
      "name": "万科金域华府",
      "cases": [
        {
          "id": "case-003",
          "title": "万科金域华府 12栋 1502室",
          "featured": true,
          "images": [
            "images/community-002/case-003/1.jpg",
            "images/community-002/case-003/2.jpg"

        }

```

**字段说明**：
- `brand`：品牌信息（Logo、名称、slogan）
- `wechat_qrcode`：微信二维码图片路径
- `communities`：小区列表
  - `id`：小区唯一标识
  - `name`：小区名称
  - `cases`：该小区的案例列表
    - `id`：案例唯一标识
    - `title`：案例标题（如"XX小区 X栋 XXX室"）
    - `featured`：是否为精选案例（`true` 表示在精选区展示）
    - `images`：该案例的效果图路径数组（按顺序排列）

## 五、本地管理界面

提供一个本地运行的 HTML 管理页面（`admin.html`），功能包括：

1. **品牌信息管理**：上传/更换 Logo、修改品牌名称和 slogan
2. **微信二维码管理**：上传/更换微信二维码图片
3. **小区管理**：
   - 添加新小区（输入小区名称）
   - 删除小区
   - 编辑小区名称
4. **案例管理**：
   - 在指定小区下添加新案例
   - 上传案例图片（支持多张批量上传）
   - 设置案例标题
   - 标记/取消精选
   - 删除案例
   - 调整图片顺序（拖拽排序）
5. **数据导出**：
   - 操作完成后，自动生成 `cases.json` 文件
   - 图片自动保存到 `/images` 对应目录
   - 用户手动执行 git add → commit → push 更新到 GitHub Pages

**管理界面技术要求**：
- 纯前端实现，数据存储在浏览器 localStorage 中
- 使用 FileReader API 读取本地图片并转为 base64 存储
- 导出时将 base64 图片保存为文件，同时生成 JSON 配置
- 无需后端，双击 `admin.html` 即可在浏览器打开使用

## 六、设计与交互规范

### 6.1 整体风格
- **建筑杂志风格**：大图、留白、精致排版，传达专业与品质感
- **色彩系统**：深墨绿主色 + 暖铜色强调色 + 纯白背景，让效果图成为主角
- **移动端优先**：主要用户通过手机扫码访问，响应式设计适配全设备
- **克制动效**：每个动画都有明确目的，避免过度装饰

### 6.2 动画与交互（与第四章保持一致）
- **品牌区（Hero）**：图片从模糊到清晰过渡（0.8s），文字淡入上浮（0.6s, delay 0.2s），底部滚动提示箭头缓慢上下浮动
- **精选案例区**：桌面端横向滚动（GSAP ScrollTrigger），移动端纵向全屏滑动，案例切换时图片滑入 + 文字淡入
- **小区选择区**：搜索框实时过滤，点击小区平滑滚动到对应案例区，卡片进入视口时淡入
- **案例详情区**：以案例为单位展示，每个案例多张图片轮播（Swiper.js），支持手势滑动，点击进灯箱，案例间用间距分隔（80-120px）
- **减少动效支持**：所有动画提供 `@media (prefers-reduced-motion: reduce)` 降级方案

### 6.3 颜色兼容性
CSS 中使用 OKLCH 色彩空间，但需提供 RGB fallback：
```css
/* Fallback first */
background-color: rgb(25, 70, 60); /* 深墨绿近似值 */
background-color: oklch(25% 0.08 160);
```

### 6.4 图片优化
- 图片懒加载（进入视口时才加载）
- 使用 `loading="lazy"` 或 Intersection Observer
- 图片使用 WebP 格式（提供 JPG 降级方案）
- Hero 背景图预处理：轻微暗角 + 低饱和度，提升文字可读性
- 缩略图使用小尺寸，点击查看大图

### 6.5 性能要求
- 首屏加载时间 < 3秒（4G 网络）
- 动画流畅，无卡顿（60fps）
- 图片按需加载，避免一次性加载所有图片
- GSAP 和 Swiper.js 通过 CDN 引入，利用浏览器缓存

## 七、技术栈建议

- **框架**：纯 HTML + CSS + JavaScript（或 Vue/React 单文件，通过 CDN 引入）
- **动画库**：GSAP / ScrollTrigger（滚动动画）或 AOS（简单滚动动画）
- **图片轮播**：Swiper.js（支持手势、响应式）
- **灯箱**：PhotoSwipe 或 Lightbox2
- **构建**：不需要构建工具，直接部署静态文件到 GitHub Pages

## 九、部署流程

1. 本地开发完成后，将代码推送到 GitHub 仓库
2. 在仓库 Settings → Pages 中启用 GitHub Pages（选择 main 分支）
3. 生成 GitHub Pages 网址（如 `https://<username>.github.io/<repo>/`）
4. 将该网址生成二维码（可使用草料二维码等工具）
5. 打印二维码，放在名片、宣传单、门店等位置

**添加新案例的流程**：
1. 双击打开 `admin.html`
2. 在管理界面中添加小区、上传案例图片、设置精选
3. 点击"导出"，自动保存图片和 JSON 到对应目录
4. 执行 `git add . && git commit -m "添加新案例" && git push`
5. GitHub Pages 自动更新，几分钟后客户即可看到新内容
